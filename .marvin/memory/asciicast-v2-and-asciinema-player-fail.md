Generating asciicast v2 for asciinema-player@3.17.0 has four failure modes that produce a broken-but-green result rather than an error — verified at source in `node_modules/asciinema-player/dist/core-*.js`, all now guarded by `packages/site/test/casts.test.mjs`. (1) The v2 header keys geometry as `width`/`height`; `parseAsciicastV2` reads exactly those, so a `cols`/`rows` header does not throw — it renders at the wrong size. Its guard is `header.width === 0 ? DEFAULT : header.width`, so `0` silently defaults (and `NaN` propagates until the player's own size validation throws), which is why the assertion must be "an integer greater than zero", not "numeric". (2) v2 event times are ABSOLUTE — `events.map(e => [e[0] * 1000, ...])` with no accumulation; v3 is the one that accumulates. A delta-encoded v2 cast plays at the wrong pace instead of failing. (3) `parseJsonl` keeps event lines with `l[0] === "["` and SILENTLY DROPS everything else, so a single indented line yields an empty recording and a blank terminal with no console output — replicate that exact predicate in a guard, because a looser "is it valid JSON" parse passes on precisely the input the player discards. (4) The player takes a recording's duration from its LAST EVENT (`const duration = events[events.length - 1][0]`), so any time padded after the final event is invisible to playback; a 1.4s trailing hold silently made every stated duration up to 19% too long. Assert `duration === lastEventTime` exactly — a `>=` assertion passes against exactly that bug. Also: the player's own `formatTime` FLOORS (`let s = Math.floor(seconds)`), so any duration you render alongside it must floor too or the two disagree by a second. Related trap in the same family: a missing vendor stylesheet is not an error either — `asciinema-player.css` absolutely positions every row and span, so without it the terminal mounts and collapses to a point, which an overflow-only assertion passes happily. Measure `boundingBox()` width AND height, not just overflow. See [[porting-a-hi-fi-html-mockup-to-an-astro]] for the overflow half.
